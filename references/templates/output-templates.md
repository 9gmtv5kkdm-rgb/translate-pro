# 输出格式（标准 / 简版 / 模式 A-D 模板）

## 📤 输出格式

### 标准输出模板

```
【原文】
（用户的原始文本）

---

【译文】
（翻译结果，直接可用）

---

【翻译说明】
- **场景**：S3（学术论文翻译）
- **主要策略**：直译为主，术语统一，结构保留
- **L层级**：达到 L3（场景地道）
- **关键决策**：
  1. [具体翻译决策和理由]
  2. [文化处理说明]
- **本地化处理**：（如适用）称呼转换/单位换算/文化适配说明
- **不确定处**：（如适用）标注哪些地方存在不确定性及原因

---
【备选译法】（如有必要）
- 方案B：[不同的翻译策略下的译法]
  - 策略：意译为主
  - 适用场景：如果……，可以这样翻
  - [备选译文]

---
【质量自评】
准确度X | 通顺度X | 地道度X | 风格度X | 实用度X | 诚实度X → 总分XX/60
```

### 简版输出模板（日常快速翻译）

当用户只是快速问"XXX用英语怎么说"等简单翻译时，使用简版：

```
【译文】
XXX

【说明】
策略：直译/意译
场景：S1（日常对话）
```

### 模式A：单句/短语翻译（快译模式）

用户问单个词、短语、短句的翻译 → 快速输出+简短说明：

```
【译文】
（翻译结果）

【说明】
直译，取自（来源/查证信息）
（如有多种译法，列出并说明各适用场景）
```

### 模式B：段落/篇章翻译（深译模式）

用户发送一段或多段文字要求翻译 → 走完整五步流程，输出标准格式。

### 模式C：翻译校对/润色

用户已有译文，要求检查或润色：

```
【原文】
…

【原译文】
…

【优化后译文】
…

【校对说明】
| # | 原译文 | 优化后 | 问题 | 类型 |
|---|--------|--------|------|------|
| 1 | [原文] | [优化] | [问题描述] | L层级问题/术语/通顺/地道 |
| 2 | … | … | … | … |

【质量提升】原译文 XX/60 → 优化后 XX/60
```

### 模式D：多语种对比

用户要求看同一内容在不同语言下的表现：

```
【原文】
…

| 语言 | 译文 | 策略 | 说明 |
|------|------|------|------|
| 英文 | … | 直译 | … |
| 日文 | … | 意译 | … |
| 韩文 | … | 本地化 | … |

【跨语言差异说明】
（指出各译本间因文化差异导致的显著不同）

【推荐】（如果用户有指定使用场景，给出最佳选择）
```
